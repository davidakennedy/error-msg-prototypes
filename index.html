New options

These examples show error message placement below the control for the CMS Design System and the Healthcare,gov Child Design System. The different options explore various ways to make the placement work for assistive technology.

Option 1

This option keeps puts the input inside the label element with both the help text and error text.

Testing notes

JAWS + Edge: Announces: Error field, Helpful hint text, Example error message, Error field, Helpful hint text edit, type in text

JAWS + IE 11: Announces: Error field, Helpful hint text, Example error message, edit, type in text

JAWS + Chrome: Announces: Error field, Helpful hint text, edit, type in text

NVDA + Firefox: Announces: Error field, Helpful hint text, Example error message, Error field, Helpful hint text edit, blank

Narrator + Edge: Announces: Error field, Helpful hint text, edit

Talkback Android + Chrome: Announces: Edit box, Error field, helpful hint text

Option 2

This moves the error message outside of the label, and below the control. It uses aria-describedby on the input to associate the error message to the input.

Testing notes

VoiceOver + Safari: If a role="alert" is on an element that has text also used as text for aria-describedby, VoiceOver will not announce the aria-describedby text. Related independent testing.

Announces: Error field, and one more item, edit text

Option 4

This option is interactive. To display it fully, click the "Submit" button without inserting anything in the field. It includes:

  • aria-invalid added to the input after an error is present.
  • Includes role="alert" on the error message.
  • Uses aria-describedby on the input to associate it to the error message.
  • Shifts focus to the error message container.

These are added to see how they work together, or not, with assistive technology.

Testing notes

VoiceOver + Safari: Announces: Error field and one more item, Invalid data, edit text

When error is triggered, the full error message is announced because focus shifts to it.

JAWS + Edge: Announces: Error field, Helpful hint text, edit

When error is triggered, the full error message is announced because focus shifts to it. Also, announces the alert role.

JAWS + IE 11: Announces: Error field, Helpful hint text, Example error message, invalid entry, text edit, type in text

When error is triggered, the full error message is announced because focus shifts to it. It announces twice, likely because of the alert role and aria-describedby.

JAWS + Chrome: Announces: Error field, Helpful hint text, Edit, Invalid entry, type in text

NVDA + Firefox: Announces: Error field, Helpful hint text, edit, Example error message, Has invalid entry, Error message, blank

Narrator + Edge: Announces: Error field, Helpful hint text, Invalid

When error is triggered, it announces Alert, Example error message

VoiceOver iOS + Safari: Announces: Error field, helpful hint text, text field, invalid data

When error is triggered, it announces Example error message

Talkback Android + Chrome: Announces: Error, invalid entry, Error field, helpful hint text, example error message

When error is triggered, it announces, Example error message, Alert, Example error message. Error announced twice, likely because of the alert role and aria-describedby.
